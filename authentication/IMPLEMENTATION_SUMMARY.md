## 🔄 Authentication Flow

### Registration Flow
1. Client sends registration request with user details
2. Server validates email uniqueness
3. Password is hashed using BCrypt
4. User is saved to database
5. Access and refresh tokens are generated
6. Tokens are returned to client

### Login Flow
1. Client sends login credentials
2. Spring Security authenticates user
3. User details are loaded from database
4. Login metadata is updated (IP, timestamp)
5. Access and refresh tokens are generated
6. Tokens are returned to client

### Protected Request Flow
1. Client includes JWT in Authorization header
2. JwtAuthenticationFilter intercepts request
3. Token is extracted and validated
4. Issuer and audience are verified
5. User details are loaded
6. SecurityContext is populated
7. Request proceeds to controller

### Token Refresh Flow
1. Client sends refresh token
2. Token validity is verified
3. IP address is checked for anomalies
4. New access token is generated
5. Same refresh token is returned
6. New access token is sent to client

## 🛠️ Configuration Options

### JWT Settings (application.properties)
```properties
# Secret key for signing tokens (base64 encoded)
jwt.secret.key=<your-secret-key>

# Access token expiration (15 minutes = 900000ms)
jwt.access.token.expiration=900000

# Refresh token expiration (7 days = 604800000ms)
jwt.refresh.token.expiration=604800000

# Issuer validation
jwt.issuer=authentication-service

# Audience validation
jwt.audience=authentication-client
```

### Database Settings
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/auth_db
spring.datasource.username=your_username
spring.datasource.password=your_password
```
